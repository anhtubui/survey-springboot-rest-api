# Using CommandLineRunner for Startup Operations

This phase focuses on performing database operations automatically when the Spring Boot application starts.

## 10. Startup Operations with CommandLineRunner

The `CommandLineRunner` is a functional interface used to run a code block **after the application context is fully initialized** but before the application starts accepting requests. It is commonly used for database seeding, warming up caches, or validating external system connections.

Common Use Cases:

- **Seeding Data**: Populating the database with default or test data.
- **Validation**: Ensuring that required external services or configurations are available.
- **Caching**: Warming up application caches before the first request arrives.
- **Startup Logging**: Printing application status or versioning info to the logs.

---

### Creating the CommandLineRunner

The `UserDetailsCommandLineRunner` class is annotated with `@Component` so that Spring detects it and executes its `run` method automatically at startup. If you have multiple runners, you can use the `@Order` annotation to control the sequence of execution.

**UserDetailsCommandLineRunner.java**:

```java
@Component
public class UserDetailsCommandLineRunner implements CommandLineRunner {

    private Logger logger = LoggerFactory.getLogger(getClass());
    private UserDetailsRepository repository;

    // Constructor Injection
    public UserDetailsCommandLineRunner(UserDetailsRepository repository) {
        this.repository = repository;
    }

    @Override
    public void run(String... args) throws Exception {
        // Save initial data
        repository.save(new UserDetails("anhtu", "admin"));
        repository.save(new UserDetails("phuongtrinh", "admin"));
        repository.save(new UserDetails("tranlebao", "user"));

        // Retrieve and log data
        List<UserDetails> users = repository.findAll();
        users.forEach(user -> logger.info(user.toString()));
    }
}
```

---

### Key Concepts

1.  **CommandLineRunner Interface**: A functional interface used to indicate that a bean should run when it is contained within a `SpringApplication`.
2.  **Constructor Injection**: The preferred way to inject dependencies (like `UserDetailsRepository`) into a component. It ensures the component is initialized with its required dependencies.
3.  **CRUD Operations**:
    - `repository.save()`: Inserts a new record into the database.
    - `repository.findAll()`: Retrieves all records from the table.
4.  **SQL Logging**:
    To see the actual SQL queries generated by Hibernate, enable the following property in `application.properties`:
    ```properties
    spring.jpa.show-sql=true
    ```
    This helps in verifying that `save()` translates to `INSERT` and `findAll()` translates to `SELECT`.

---

### Summary of Startup Flow

1. Spring Boot Application Starts.
2. Application Context is initialized.
3. Hibernate creates/updates the database schema.
4. `CommandLineRunner` beans are executed.
5. SQL queries are logged to the console (if enabled).
